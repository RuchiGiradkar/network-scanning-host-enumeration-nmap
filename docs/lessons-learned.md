# Lessons Learned

## Technical Lessons

* UDP scanning is significantly slower than TCP scanning.
* Open|filtered states require careful interpretation.
* Automated vulnerability scans can generate false positives.
* OS fingerprinting should be correlated with service enumeration.
* Router interfaces often expose minimal but important services.

## Security Lessons

* Exposure does not automatically equal vulnerability.
* Validation is a critical step in vulnerability management.
* Security assessments require both technical evidence and analyst judgment.
* Documentation is as important as discovery.

## Professional Lessons

This project reinforced the importance of combining technical findings, validation activities, risk assessment, and remediation planning into a structured security assessment workflow.
